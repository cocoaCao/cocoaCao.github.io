1.4 列表

Markdown 支持无序列表和有序列表。

1.5.1 无序列表

无序列表使用*、+或是-作为列表标记：

  • Red
  • Green
  • Blue
  • Red
  • Green
  • Blue
  • Red
  • Green
  • Blue

    1.4.2 有序列表

    有序列表则使用数字接着一个英文句点：
  1. Bird
  2. McHale
  3. Parish

值得一提的是，在有序列表中使用的数字并不会影响输出的 HTML 结果：

  1. Bird
  2. McHale
  3. Parish

或

  1. Bird
  2. McHale
  3. Parish

1.5 代码

Markdown标记代码有缩进用` 符号放在代码前后，或使用 ```来包含多行代码

1.5.1 缩进方式

代码块前后需要有至少一个空行，且每行代码前需要有至少一个 Tab 或四个空格：

这是一个普通段落：

这是一个代码区块。
 这是一个代码区块。
 这是一个代码区块。

这是一个代码区块。

可以注意到缩进方式标记代码，代码行的缩进会被移除，一个代码区块会一直持续到没有缩进的那一行（或是遇到空行）。

1.5.2 使用 ` 符号放在代码前后

code
如果要在代码区段内插入` ，你可以用多个` 来开启和结束代码区段：

There is a literal backtick (`) here.

1.5.3 使用 ``` ``` 来包含多行代码

注意在第一组 ``` 之后可以添加代码的语言

1.6 分割线

可以在一行中使用三个或更多的 *、- 或 _ 来添加分隔线

多个字符之间可以有空格（空白符），但不能有其他字符.

1.7 超链接

Markdown中实现超链接有3中：行内式、参考式和自动连接

1.7.1 行内式

行内式格式如下：

This is an example inline link.
This link has no title attribute.

注：如果带有’title’,鼠标移到超链接上，会显示title内容

假如是指向本地的连接：

icon.png

1.7.2 参考式

参考式链接的写法相当于行内式拆分成两部分，并通过一个 识别符 来连接两部分。参考式能尽量保持文章结构的简单，也方便统一管理 URL。

This is an example reference-style link.

注：
a. 链接的定义可以放在文件中的任何一个地方
b. 下面三种方式相同
[id]: http://example.com/ "title"
[id]: http://example.com/ 'title'
[id]: http://example.com/ (title)

2.2 表格

2.2.1 单元格和表头

使用 | 来分隔不同的单元格，使用 - 来分隔表头和其他行：

name age
LearnShare 12
Mike 32

为了美观，可以使用空格对齐不同行的单元格，并在左右两侧都使用 | 来标记单元格边界：

name age
LearnShare 12
Mike 32

为了使 Markdown 更清晰，|和- 两侧需要至少有一个空格（最左侧和最右侧的|外就不需要了）。

2.2.2 对齐

在表头下方的分隔线标记中加入 :，即可标记下方单元格内容的对齐方式：

left center right
aaaa bbbbbb ccccc
a b c

2.2.3 插入其他内容

表格中可以插入其他 Markdown 中的行内标记：

name age blog
CocoaCao 12 CocoaCao
简书 32 简书
